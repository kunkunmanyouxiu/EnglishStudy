## 1. 功能介绍

### 核心功能

| 功能 | 说明 |
|------|------|
| **双向默写** | 英文→中文、中文→英文两种模式，随机抽题 |
| **AI 智能判题** | 本地运行 Gemma 4 E4B 模型，语义理解判断答案是否正确，而不仅仅是字符串匹配 |
| **科学复习** | 基于艾宾浩斯遗忘曲线自动安排复习计划，到期自动提醒 |
| **自定义词库** | 创建个人词库，手动添加或批量导入单词 |
| **AI 词库校验** | 上传自定义单词前，AI 自动检查英文单词与中文释义是否匹配 |
| **错题本** | 自动收录答错单词，支持专项练习 |
| **学习统计** | 展示累计学习数据和今日默写结果分布图 |
| **单词发音** | 内置 TTS 英文发音（美式），默写时可点击朗读 |

### AI 判题说明

判题采用**两阶段机制**：

1. **快速通道（无需 AI，毫秒级）**：
   - 答案与标准释义精确匹配 → 直接判"正确"
   - 中→英模式单字符拼写错误 → 直接判"部分正确"

2. **AI 语义判断**：
   - 精确匹配无法判断时，调用本地 Gemma 4 E4B 模型进行语义理解
   - 例如：标准释义"放弃"，用户输入"抛弃"→ AI 判断语义一致 → "正确"

### 内置词库

应用内置以下标准词库分类（需自行导入单词）：

- 四级、六级、考研、雅思、托福

## 3. 下载并配置 AI 模型

> 这是运行本应用**最关键的一步**，请务必按照步骤操作。

### 第一步：下载模型文件

模型托管在 Hugging Face，文件名为 `gemma-4-E4B-it.litertlm`，大小约 **3.65 GB**。

**方式一：直接下载（推荐）**

访问以下链接，点击文件名旁的下载按钮：

```
https://huggingface.co/litert-community/gemma-4-E4B-it-litert-lm](https://huggingface.co/litert-community/gemma-4-E4B-it-litert-lm/tree/main
```

进入页面后，点击顶部的 **"Files and versions"** 选项卡，找到 `gemma-4-E4B-it.litertlm`（3.65 GB），点击下载箭头。


### 第二步：将模型文件放入 assets 目录

下载完成后，将文件复制到以下路径：

```
项目根目录/
└── app/
    └── src/
        └── main/
            └── assets/
                └── gemma-4-E4B-it.litertlm   ← 放在这里
```

**在 Android Studio 中操作：**

1. 在左侧 Project 面板切换到 **"Android"** 视图
2. 展开 `app` → `src` → `main`
3. 如果没有 `assets` 文件夹，右键 `main` → **New → Folder → Assets Folder**
4. 将下载好的 `gemma-4-E4B-it.litertlm` 复制（或拖入）到 `assets` 文件夹下

**确认路径正确：** 在文件系统中，完整路径应为：

```
<项目路径>\app\src\main\assets\gemma-4-E4B-it.litertlm
```

### 第三步：确认代码中的文件名

打开 `app/src/main/java/com/example/myapplication/ai/ModelManager.kt`，确认第 18 行的文件名与下载的文件完全一致：

```kotlin
private const val MODEL_FILENAME = "gemma-4-E4B-it.litertlm"
```

> ✅ 当前代码中的文件名已与 Hugging Face 上的实际文件名保持一致，**无需修改**。

## 4. 编译与安装

### 编译

1. 用 Android Studio 打开项目根目录
2. 等待 Gradle 同步完成
3. 菜单栏 → **Build → Make Project**，确认无编译错误

### 安装到手机

> ⚠️ 由于 assets 中包含 3.65 GB 的模型文件，APK 体积极大，**通过 USB 安装耗时较长（约 20~40 分钟）**，请耐心等待，这是正常现象。

1. 手机开启**开发者选项** → 打开 **USB 调试**
2. 用 USB 数据线连接手机（建议使用 USB 3.0 接口）
3. Android Studio 顶部工具栏选择目标设备
4. 点击绿色三角形 ▶ **Run** 按钮
5. 等待 Android Studio 底部状态栏显示 **"Connected"**（而非一直显示 "Launching"）

**加速安装提示：**
- 使用 USB 3.0 线和接口（传输速度是 USB 2.0 的 10 倍）
- 确保手机存储有足够空间

## 5. 首次启动

### 模型初始化（必须等待）

首次启动时，应用会自动将模型从 APK 的 assets 目录**复制到手机内部存储**，这一过程：

- 需要约 **5~15 分钟**（取决于手机存储速度）
- 期间主界面顶部显示：**"🚀 模型加载中..."**
- **此时仍可正常使用**：词库管理、导入单词等功能不受影响

复制完成后，界面显示 **"✅ AI 模型已就绪"**，即可开始智能默写。

> **重要：** 模型只在**首次启动**时复制，之后每次启动直接从内部存储加载，速度很快（通常 30 秒~2 分钟）。

### 导入单词

应用首次运行时词库为空，需要先导入单词才能使用默写功能。请参考第 6 节的导入说明。

## 6. 功能使用说明

### 6.1 导入单词

#### 方式一：CSV 批量导入（推荐）

1. 主界面点击 **"自定义词库"**
2. 点击右上角 **"📥 导入"** 按钮
3. 选择 CSV 文件（格式见[第 7 节](#7-csv-词库格式说明)）
4. 在弹出的对话框中选择导入到哪个词库（或新建词库）
5. 点击 **"✅ 开始导入"**，等待进度完成

#### 方式二：手动逐条添加

1. 主界面点击 **"自定义词库"**
2. 进入某个词库，点击 **"词库管理"** 或右下角 **"+"** 按钮
3. 填写：英文单词、音标（可选）、中文释义、例句（可选）
4. 点击 **"AI 预判断"**，等待 AI 检查匹配度：
   - **高**：点击"立即上传"
   - **中**：查看优化建议，可选择修改后重新判断，或直接上传
   - **低**：建议修改释义后重新判断
5. 点击 **"立即上传"** 完成添加

---

### 6.2 管理自定义词库

1. 主界面点击 **"自定义词库"**
2. 可进行以下操作：

| 操作 | 步骤 |
|------|------|
| 新建词库 | 点击 **"+ 新建"** → 输入名称 → 创建 |
| 重命名词库 | 点击词库卡片上的**改名按钮** → 输入新名称 |
| 删除词库 | 点击词库卡片上的**删除按钮** → 确认（单词数据保留）|
| 查看词库单词 | 点击词库卡片 → 跳转到词库详情 |

---

### 6.3 开始默写

1. 主界面点击 **"英文→中文默写"** 或 **"中文→英文默写"**
2. 在弹出的**词库选择对话框**中选择要默写的词库
3. 进入默写界面后：

**英文→中文模式：**
- 屏幕上方显示英文单词和音标
- 在输入框中填写中文释义
- 点击 **🔊** 可朗读单词

**中文→英文模式：**
- 屏幕上方显示中文释义和例句
- 在输入框中填写英文单词

4. 填写完成后点击 **"提交"**，查看判题结果：
   - 🟢 **绿色"正确"**：3 秒后自动跳到下一题
   - 🟠 **橙色"部分正确"**：查看标准释义，3 秒后自动跳题
   - 🔴 **红色"错误"**：查看标准释义和提示，点击 **"重新默写"** 重试

5. 默写过程中可点击右上角 **"选择词库"** 切换词库

---

### 6.4 今日复习

1. 主界面点击 **"今日复习（艾宾浩斯）"**
2. 查看今日待复习词数
3. 点击 **"开始复习"** 进入复习默写（流程同普通默写）

> **复习原理：** 应用根据艾宾浩斯遗忘曲线，自动计算每个单词的复习时间。  
> 答对 → 延长间隔（最长 60 天）；答错 → 缩短间隔，优先复习。

**复习间隔示例（"掌握"状态）：**

```
第 1 次答对 → 7 天后复习
第 2 次答对 → 14 天后复习
第 3 次答对 → 30 天后复习
第 4 次以上 → 60 天后复习
```

---

### 6.5 错题本

1. 主界面点击 **"错题本"**
2. 查看所有答错记录，包含：
   - 单词和释义
   - 错误次数和上次错误答案
3. 点击 **"开始错题练习"** 进行专项练习
4. 已掌握的单词可点击 **"移除"** 从错题本中删除

---

### 6.6 词库管理（浏览）

1. 主界面点击 **"词库管理"**
2. 顶部 Chip 可切换分类：全部 / 四级 / 六级 / … / 自定义词库
3. 顶部搜索框实时搜索（支持按单词或释义搜索）

---

### 6.7 学习统计

1. 主界面点击 **"学习统计"**
2. 查看以下数据：
   - 词库总量 / 自定义单词数 / 词库数
   - 累计学习次数
   - 今日已复习词数
   - 错题总数
   - 今日正确 / 部分正确 / 错误分布（柱状图）

## 7. CSV 词库格式说明

### 文件要求

- 编码：**UTF-8**（不带 BOM）
- 格式：标准 CSV，**首行为表头**（会被跳过）
- 文件扩展名：`.csv`

### 列格式

```
word,phonetic,definition,type
```

| 列名 | 说明 | 是否必填 |
|------|------|---------|
| `word` | 英文单词 | ✅ 必填 |
| `phonetic` | 音标（如 `[əˈbændən]`） | 可选（留空即可） |
| `definition` | 中文释义 | ✅ 必填 |
| `type` | 词库分类（通常填词库名） | 可选 |

### 示例文件

```csv
word,phonetic,definition,type
abandon,[əˈbændən],放弃；遗弃,四级
ability,[əˈbɪlɪtɪ],能力；才能,四级
absence,[ˈæbsəns],缺席；缺乏,四级
absolute,[ˈæbsəluːt],绝对的；完全的,四级
achieve,[əˈtʃiːv],实现；达到,六级
acknowledge,[əkˈnɒlɪdʒ],承认；致谢,六级
```

> **提示：** 如果 phonetic 列为空，可以写成 `abandon,,放弃；遗弃,四级`（中间两个逗号）。

## 8. 常见问题

### Q：安装时 Android Studio 一直显示 "Launching" 怎么办？

**原因：** APK 中包含 3.65 GB ���模型文件，通过 USB 传输需要较长时间。

**解决方法：**
- 耐心等待
- 确保 USB 线和接口支持 USB 3.0 以上规格
- 不要断开 USB 连接，等待 Android Studio 底部显示 "Connected"

---

### Q：进入应用后一直显示 "AI 模型���载中" 怎么办？

**原因：** 首次启动时，应用正在将模型从 APK 复制到内部存储（3.65 GB，约需 5~15 ���钟）。

**解决方法：**
- 保持应用在前台运行，耐心等待
- 期间可以先进行词库管理和导入操作
- 复制完成后自动显示 "✅ AI 模型已就绪"

---

### Q：提示 "AI 模型尚未就绪" 无法判题怎么办？

等待首次模型加载完成后即可正常使用。若多次重启后仍提示，请检查：
1. `assets` 目录中模型文件是否存在且完整（大小应为 3.65 GB）
2. 手机内部存储空间是否足够（至少需要 4 GB 空余）

---

### Q：默写选择词库时列表为空怎么办？

需要先导入单词。请参考 [6.1 导入单词](#61-导入单词) 章节，通过 CSV 导入或手动添加的方式先添加单词。

---

### Q：CSV 导入后单词没有出现？

请检查：
1. CSV 文件编码是否为 **UTF-8**（不是 GBK 或 ANSI）
2. 首行是否为表头（`word,phonetic,definition,type`），导入时会自动跳过首行
3. `word` 和 `definition` 列是否为��

---

### Q：判题速度很慢怎么办？

精确匹配的情况（答案与标准释义完全一致）无需 AI，响应是毫秒级的。

当答案与标准释义不完全一致时，需要 AI 进行语义判断，速度取决于设备性能：
- 高端旗舰机（骁龙 8 Gen 3 / 天玑 9300 以上）：约 3~8 秒
- 中端设备：约 10~20 秒
- 入门级设备：可能需要更长时间

---

### Q：模型加载失败，显示 "模型文件太小" 怎么办？

说明 assets 中的模型文件不完整。请重新下载 `gemma-4-E4B-it.litertlm`，确保文件大小为 **3.65 GB**，再重新编译安装。
